
1. SW Life Cycle
  1) 센서(아두이노)는 주변의 상황을 체크하여 서버로 전송한다.(WIFI 통신)
  2) 서버(라즈베리파이)는 센서로부터 상태 정보를 전달받는다.
  3) 전달받은 정보를 바탕으로 평시 / 화재상황을 판단한다.
  4) 화재상황 발생시 모든 View를 대피경로 안내 화면으로 변경한다.
  5) 센서로부터 변동사항을 실시간으로 전달받아 대피경로를 실시간으로 안내한다.
    * 화재상황에서는 데이터 전송 주기를 짧게한다.
  6) 모든 경로가 사용 불가능일 경우 가장 안전한 노드로 안내한다.

2. 개인별 파트

  1) 하드웨어 : 정지훈
    - 네트워크 통신(데이터 전송)구상
    - 아두이노 프로그래밍
    - 센서 회로

  2) 웹 서버 : 김동현
    - View 구상 ( 스토리 보드 )
    - node.js express이용 웹 서버 구축
    - 네트워크 통신(데이터 받기)구상

  3) 서비스 : 김용재
    - node-dijkstra NPM 사용
    - https://www.npmjs.com/package/dijkstra
    - 최단 경로 탐색 알고리즘 적용
    - 센서정보를 통해 경로를 재탐색하는 메소드 구현

  4) DB : 권수정
    - JSON 형식
    { 'A' : {  'B'   : 1  },  'B' : {  'A'   :  1  ,  'C'   :   2 ,   'D'  :  4  } }
    { 노드 : {연결노드 : 거리}, 노드 : {연결노드 : 거리 , 연결노드 : 거리, 연결노드 : 거리} }

    - DAO 인터페이스 구상
